## 1. 신규 기능

### 생산지시 목록 — 체크박스 드래그 선택
- 미배정 대기 목록, 설비배정 완료 목록 둘 다 체크박스를 하나씩 클릭하는 대신 마우스로 쭉 긁어서 여러 건을 한 번에 선택할 수 있게 함 (60건/일 이상 물량 대응)

### 설비배정 완료 — 일괄 확정
- 기존엔 배정된 건을 하나씩 "확정" 버튼으로 눌러야 했는데, 여러 건 체크 후 "선택 항목 일괄 확정" 한 번에 가능하도록 추가 (기존 일괄배정과 동일한 패턴)

### 가디자인 첨부 — 수주 라인(품목)별로 분리
- 원래는 수주 하나에 첨부한 가디자인 파일이 그 수주의 **모든 품목에 공통으로** 적용되는 구조였음 (품목이 여러 개인 수주에서 잘못된 파일이 섞여 보임)
- 수주 품목 목록의 각 행마다 개별 📎 첨부 버튼을 추가해서, 품목별로 별도 파일을 붙일 수 있도록 구조 변경

### 품목등록 — 단위/수량 표기 정리
- "완제품단위"(실제로는 1롤당 매수 같은 환산수량)와 "단위"(주문단위: 롤/박스 등)가 라벨링이 헷갈리게 붙어있던 것을 정리
- 순서를 "단위 선택 → 그 단위당 매수" 흐름으로 바꾸고, 라벨도 선택한 단위에 따라 동적으로 바뀌게 함 (예: "1롤당 매수")
- 판매단가 입력란이 화면에서 완전히 숨겨져 있던 것도 다시 노출 (총단가 자동계산 추가)

### 수주등록 — 수주수량의 매/단위 환산 표시
- 수주수량은 항상 "매" 기준으로 입력하는데, 옆에 "= 3.5롤"처럼 실제 주문단위 환산값을 자동으로 보여주도록 추가 (매수인지 롤수인지 헷갈리던 문제)

## 2. 발견 및 수정한 버그

### 체크박스 컬럼 추가 이후 조용히 깨져있던 기능 6곳
품목 목록 테이블 맨 앞에 일괄선택용 체크박스 칸이 추가되면서, "행의 첫 번째 칸 = 품목코드"라고 가정하던 예전 코드들이 전부 빈 값을 읽고 있었음. 아래 6곳이 전부 이 이유로 깨져 있었음:
1. **고객사코드 컬럼** — 뭘 입력해도 화면엔 항상 "－"로 표시 (데이터 자체는 저장되어 있었음)
2. **품목 수정 저장** — 수정 화면에서 저장해도 실제로는 아무것도 갱신 안 됨 (가장 심각했던 문제)
3. **디자인 등록요청 상태 뱃지** — 요청/등록 상태로 안 바뀜
4. **인쇄유무(P/M) 최종 확정 로직** — 디자인 등록 완료 후 품목코드의 "미정" 표시가 P/M으로 안 바뀜
5. **BOM 등록 화면의 "완제품 선택" 드롭다운** — 옵션에 품목코드가 비어서 나와 사실상 사용 불가
6. **수주등록 "품목 불러오기"의 "디자인 등록완료만" 필터** — 항상 0건만 나옴

### 품목등록 — 삭제 버튼이 가짜였음
- "삭제" 버튼을 눌러도 확인창+알림만 뜨고 실제로는 아무 데이터도 안 바뀌었음 → 실제로 "사용유무=단종" 처리하고 저장·화면 갱신까지 하도록 구현

### 수주등록 "품목 불러오기" — 개별 등록 품목이 안 보이던 문제 (2건)
1. "완제품만" 체크박스가 기본으로 켜져있어서, 완제품이 아닌 개별 등록 품목(상품 등)이 기본으로 안 보임 → 기본값 해제, 원자재/부자재는 애초에 주문 대상이 아니므로 항상 제외
2. "디자인 등록완료만" 체크박스가 한 번 켜면 팝업을 닫아도 계속 켜진 채로 남아있어서 원인 모르게 품목이 안 보이는 상황 발생 → 팝업 열 때마다 항상 꺼진 상태로 초기화, 결과 0건일 때 어떤 필터 때문인지 메시지로 안내

### 생산지시 — 신규 주문도 자동으로 설비배정 완료 상태였던 문제
- "재판이면 자동으로 설비 배정"이라는 의도로 만든 로직인데, 실제 코드에는 재판인지 확인하는 조건이 빠져있어서 **신규 주문도 무조건 자동 배정**되고 있었음
- 신규는 원래 의도대로 "미배정 대기"에 남고, 재판만 자동 배정되도록 조건 추가

### 작업지시서 출력 — 유령 데이터 표시 문제
- 생산지시(WO)는 새로고침하면 초기화되는데, 설비배정 기록(_assigns)은 IndexedDB에 계속 남아있어서, 예전에 테스트했던 배정 기록이 실제로는 존재하지 않는 지시번호를 가리키며 계속 쌓여있었음
- 생산지시 목록 화면은 "이 배정이 실제로 지금 있는 지시인지" 확인하는 로직이 있었는데, 작업지시서 출력 화면엔 그 확인이 빠져있어서 유령 데이터가 그대로 인쇄 목록에 나왔음 → 동일한 확인 로직 추가

### 디자인 등록 — 버전과 디자인명이 따로 놀던 문제
- 재판/수정을 해도 버전이 자동으로 안 올라가고(항상 v1.0), 디자인명도 버전과 무관하게 안 바뀌었음
- 기존 디자인(베이스) 선택 시 다음 버전 번호를 자동 제안(v1.0→v1.1)하고, 디자인명 끝에 붙은 버전 표시도 함께 갱신되도록 함. 완전히 다른 이름을 직접 지은 경우는 건드리지 않음

## 3. 데모/시연용 샘플 데이터 추가

- "재판(베이스 디자인 있음)" 시연을 위해 전용 데모 품목 2건을 새 코드로 추가 (`[데모] 라벨이지 롤라벨 A4`, `[데모] 라벨이지 롤라벨 100×100`)
- 각각 이미 승인완료 상태인 디자인을 미리 등록해뒀고, 로고 이미지도 디자이너 시안/품목 원본 가안 양쪽에 첨부해둠
- 실제 신규 등록 → 승인 → 재판 흐름도 처음부터 끝까지(거래처 등록~생산지시까지) 직접 클릭해서 문제없이 되는 것 확인함
