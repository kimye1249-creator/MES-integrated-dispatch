# 구매 부서 화면 설계 (2026-07-20)

## 요구사항 (김예람)
1. 구매 프로세스 인터페이스 구축
2. 입고현황조회
3. 자재입고 마감 처리 (건별 확정 → 월마감)
4. 제조경비 / 일반경비 구분

## 결정사항
- 마감 단위: **건별 확정 → 월마감** 2단계
- 경비구분: **입고 등록 폼에 필드만 추가** (별도 경비등록 화면 없음, 발주 화면 미변경)
- 구매 프로세스: **기존 purflow(구매 플로우 테스트) 화면 정식화** (신규 화면 없음)

## 구현 내용 (index.html 단일 파일)

### 1. 구매 프로세스 정식화
- 메뉴/헤더/screenSpecs에서 "구매 플로우(테스트)" → "구매 프로세스". 로직 변경 없음.

### 2. 입고현황조회 (scr-stockinlist 승격)
- 메뉴/화면명 "입고 완료 목록" → "입고현황조회"
- 필터: 월 선택(전체/월별), 경비구분(전체/제조경비/일반경비)
- 요약 스트립: 건수 · 총 입고금액 · 제조경비 합계 · 일반경비 합계
- 컬럼 추가: 경비구분, 마감상태(미확정/확정/월마감)

### 3. 마감 처리
- 데이터: `MAT_IN[].st` ('':미확정, 'ok':확정), `MAT_IN[].exp` (경비구분), `window.MAT_CLOSE` (마감월 배열, 'YYYY-MM')
- 행별 [확정]/[취소] 버튼. 월마감된 건은 취소 불가.
- 월 필터 선택 시 [월마감]/[마감해제] 버튼 활성. 월마감 시 미확정 건 있으면 확인 후 일괄 확정 처리.
- `siSave`: 마감월 날짜로 신규 입고 등록 차단.
- 영속화: 기존 `_GG1_save`/`_GG1_load` 패턴으로 MAT_IN, MAT_CLOSE 저장/복원.
- 연도: 데모 데이터 관행에 따라 2026 고정 (r.date는 'MM/DD' 형식).

### 4. 경비구분
- 입고 완료 등록 폼에 경비구분 셀렉트(제조경비/일반경비, 기본 제조경비) 추가 → MAT_IN 저장 → 조회 화면 필터·집계.

## 제외 (YAGNI)
- 발주 화면 경비구분, 별도 경비등록 화면, 회계 전표 연동, 연도 셀렉트.

## 후속: 메뉴 정리 — 부자재 전용 발주 (2026-07-21)

### 배경
- BOM은 원단만 포함하기로 결정. "자재 발주(수주)"(somatorder)는 수주 BOM 전개 기반 원단 발주라 그대로 유지.
- 박스·지관·테이프 등 BOM 미포함 부자재는 별도 구매 화면 필요.

### 결정
- 신규 화면 만들지 않음. 기존 "자재 발주"(stockorder, 수동 발주 폼)는 원래도 원자재+부자재 모두 되는 범용 폼이었으므로, **부자재 전용으로 스코프만 좁힘**.
- `soIsMat()` 필터: `type==='원자재'||'부자재'` → `type==='부자재'`만.
- 화면명/메뉴명: "자재 발주"→"부자재 발주", "자재 발주 목록"→"부자재 발주 목록" (screenSpecs·menu·mesflow SVG·MAP 동기화).
- MAT_PO/soRecentBody 데모 시드에서 원단(원자재) 발주 예시 제거, 부자재 예시로 교체 (데모 일관성).
- "자재 발주(수주)"/"자재 발주 목록(수주)"은 완전 미변경.
- 부자재 재고 확인은 기존 "부자재 현황"(substock) 화면 그대로 사용.
