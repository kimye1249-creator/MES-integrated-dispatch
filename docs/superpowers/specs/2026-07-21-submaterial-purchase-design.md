# 부자재 구매 프로세스 (2026-07-21)

## 배경
- 회사 구매부는 역할이 나뉨: BOM(원단)을 담당하는 사람과, 그 외 부자재(박스·지관·테이프 등)를 담당하는 사람.
- 원단 관련 화면("자재 발주(수주)", "자재 발주 목록(수주)", 기존 "자재 발주"/"자재 발주 목록" 등)은 이미 상세하게 잡혀있어 손대면 안 됨.
- 이 세션 초반에 기존 화면(자재 발주/목록, 입고 완료 목록)을 부자재 전용으로 재활용하려 했으나(`9b2a547`, `7209f10`), 그 방식은 원단 트리와 데이터/화면을 공유해 리스크가 있어 **전면 되돌림**(`git revert`)하고 완전히 새로운 화면으로 다시 만듦.

## 결정사항
- 기존 화면·데이터(MAT_PO, MAT_IN, CI_MASTER, MAT_STOCK 등)는 **일절 수정하지 않음**.
- 새 메뉴 그룹 **"부자재 관리"** 신설, 완전히 독립된 데이터(`SUBMAT_PO`, `SUBMAT_IN`, `SUBMAT_CLOSE`)로 부자재 구매 프로세스 구현.
- 분할입고 미지원 — 발주 1건은 입고 1회로 종결(발주수량 그대로 입고).
- 기존 "입고 완료 등록" 같은 별도 등록 화면 없이, 발주 목록에서 바로 입고 처리.

## 화면 구성

### 1. 부자재 발주 (scr-submatorder)
- 목록이 메인 화면 (기존 자재 발주 목록과 유사한 레이아웃).
- "+ 발주 등록" 버튼 → 모달(발주일자/납기/담당자/비고 + 품목 라인, 자재 검색은 `CI_MASTER`에서 `type==='부자재'`만 조회 — 읽기 전용 참조).
- 저장 시 `SUBMAT_PO`에 추가.
- 목록 행(발주중 상태)에 **[입고]** 버튼 → 미니 모달(입고일·Lot번호·창고·수입검사·경비구분 5개만 입력) → 저장 시 발주의 모든 품목 라인을 `SUBMAT_IN`에 반영 + 발주 상태 '입고완료'.
- 주문서 보기는 기존 `poDocShow()`(순수 렌더 함수, 원단 트리와 무관하게 이미 여러 데이터소스가 공용으로 쓰는 뷰어)를 재사용.

### 2. 마감 처리 (scr-submatclose)
- `SUBMAT_IN` 기준 월/경비구분 필터, 요약(건수·총액·제조/일반경비), 건별 확정→월마감.
- 로직은 이전에 "입고현황조회"에 구현했던 것과 동일한 패턴(되돌려진 버전)을 `SUBMAT_IN`/`SUBMAT_CLOSE` 대상으로 재구현.
- 영속화는 기존 `_GG1_save`/`_GG1_load`(IndexedDB) 패턴에 새 키(`SUBMAT_PO`/`SUBMAT_IN`/`SUBMAT_CLOSE`)로 저장 — 인프라만 재사용, 데이터는 완전 분리.

## 제외 (YAGNI)
- 부자재 재고 현황(기존 "부자재 현황" 화면)과의 실시간 재고 연동 — 필요 시 별도 요청.
- 발주 등록 모달의 원단폭 컬럼 — 부자재는 폭 개념이 거의 없어 생략.
- 분할입고, 발주 취소/수정 기능.

## 미결 (이전 대화에서 보류)
- "구매요청 확인" 삭제 여부, "제조 BOM 생성"/"자재 출고"를 생산 관리로 이동할지 — 그대로 둠.
